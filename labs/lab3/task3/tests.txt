- шифрование при записи
- дешифрование при чтении
- компрессия при записи
- декомпрессия при чтении

для заголовочной функции:
1. На входе неправильное количество параметров: нет выходного или входного файла
2. На входе не двоичный файл
3. Нет никаких операций над файлом -- входной и выходной файл должны совпасть

для тестирования потоков в памяти просто заменить слово File на Memory
для декораторов:
4. Простейшие опции: шифрование и дешифрование файла одной командой
4.1 Файл считывается и шифруется. То есть, входной файл и результат не совпадают, при этом результат - не пустой файл
4.2 Шифрованный файл считывается и дешифруется. Входной файл и результат не совпадают, при этом результат - не пустой файл
4.3 Файл считывается, шифруется и дешифруется. Входной файл и файл-результат должны совпасть
4.4. Файл считывается и шифруется несколько раз. Затем он дешифруется такое же количество раз. Первоначальный файл должен совпасть с файлом-результатом.
4.5 1 файл шифруется 2 ключами. 2 результата сравниваем, они должны быть разными

5. Компрессия и декомпрессия файла 1 командой
5.1 Файл считывается и сжимается. Входной файл и результат различаются, при этом файл-результат не пустой
5.2 Сжатый файл считывается и разжимается. Входной файл и результат не совпадают, при этом результат - не пустой файл
5.3 Файл считывается, сжимается и разжимается. Входной файл и файл-результат должны совпасть
5.4. Файл считывается и сжимается несколько раз. Затем он разжимается такое же количество раз. Первоначальный файл должен совпасть с файлом-результатом.

6. Проверка сложного набора опций - файл считывается, шифруется, сжимается, затем разжимается и дешифруется. Файл-результат и входной файл должны совпасть

для потоков:
1. Поток памяти считывает данные
2. Поток файлов считывает данные
3. Поток памяти выводит данные
4. Поток файлов выводит данные
5. Поток файлов выводит сообщение об ошибке при открытии файла на чтение
6. Поток файлов выводит сообщение об ошибке при открытии файла на запись
